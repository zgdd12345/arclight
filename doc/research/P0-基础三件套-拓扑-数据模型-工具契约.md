# P0 施工图：拓扑决策 + 数据模型 + 工具执行契约

> arclightagent 阶段一(P0)的"施工图级"基础三件套。由 Codex(gpt-5.5/xhigh) 读 4 份文档 + 仓库实现产出，与 `ARCHITECTURE_BLUEPRINT.md` / `FULL_PLATFORM_DESIGN.md`(§2 协议/ArcEvent) / `research/拿来即用-全栈选型清单.md`(Bun+Hono+drizzle) / `research/P0-沙箱方案-拿来即用.md`(nono SandboxService) 严格一致。
> 技术栈：Bun + Hono + drizzle-orm + bun:sqlite。P0 = 本地优先个人版，单用户但全表带 `tenant_id`。

## A) 拓扑决策（拍板，解决"Web 本地沙箱"矛盾）

**P0 = 本地优先个人版**：`arclight serve --repo <path>` 在 localhost 同时托管 Hono 内核 API/SSE + Web UI + 本地 nono 沙箱 + SQLite（`.arclight/arclight.sqlite`）。

```bash
arclight serve --repo /Users/me/project/foo
# Arclight local server: http://127.0.0.1:43127
```
- `POST /api/commands`（C1）｜`GET /api/sessions/:id/events`（C2 SSE）｜`GET /`（Web UI）｜`.arclight/`（artifacts/audit/checkpoints）
- **矛盾澄清**：P0 Web 的 `CapabilityProfile.localSandbox = true`（经本地 Hono 内核 + nono 间接具备，`fileSystem:"native"` 由内核代理）。之前 `Web localSandbox:false` **只适用于远程部署 Web**（浏览器连 VPS，本机 workspace 不在内核侧）——P0 不走该拓扑。
- **鉴权**：不上 Auth.js。loopback bearer token + httpOnly cookie（首次 `?pair=<one-time>` 配对）；CLI/测试用 `Authorization: Bearer`。
- **server.json**（`~/.config/arclightagent/server.json`，`chmod 0600`，校验 owner+mode）：`{version,pid,port,origin,token,workspaceId,repoPath,createdAt}`；**不写** provider key/MCP token/credential-proxy secret。只监听 `127.0.0.1`；CORS 默认关（dev 仅允 `127.0.0.1:*`/`localhost:*`）；token 随重启轮换。

## B) P0 数据模型（drizzle + bun:sqlite）

约定：`tenant_id="local"`、`user_id="local-user"`；`epoch`=会话乐观锁+压缩边界；`seq`=per-session 单调事件序；JSON 用 `text({mode:"json"})`；时间 Unix ms。

```ts

}, t => ({
  tenantRepoIdx: uniqueIndex("workspaces_tenant_repo_uq").on(t.tenantId, t.repoPath),
  tenantIdx: index("workspaces_tenant_idx").on(t.tenantId),
}));

export const sessions = sqliteTable("sessions", {

  createdAt: integer("created_at", { mode: "timestamp_ms" }).notNull().default(nowMs),
}, t => ({
  commandUq: uniqueIndex("turns_session_command_uq").on(t.sessionId, t.commandId),   // 幂等

export const events = sqliteTable("events", {
  id: text("id").primaryKey(),
  tenantId: text("tenant_id").notNull().default("local"),
```

**seq 生成（必须在一个 SQLite 事务内）**：读 `sessions.nextSeq`+`epoch` → insert `events(seq,epoch,...)` → update `sessions.nextSeq=seq+1, lastEventSeq=seq`。SSE `id:` 直接用 `seq`。

**event 持久化边界**：持久化所有 UI 重建所需事件（session.started/turn.started/message.delta[100-250ms 合批]/tool.requested/tool.progress[chunk 合批，超大只落 artifact]/tool.output/permission.ask/context.compacted/turn.completed/session.error/interrupted）；**不持久化纯心跳** `: heartbeat`。

**恢复/续接** `GET /api/sessions/:id/events?afterSeq=N&epoch=E`：① 查当前 epoch；② 请求 epoch 旧且 afterSeq 早于最近 `context.compacted` → `resync`/`409{reason:"epoch-jump",snapshotUrl}`；③ afterSeq 仍在表 → replay `seq>afterSeq`；④ 缺口 → `resync{reason:"buffer-expired"}`；⑤ P0 不清理当前 session events，server 重启后可从 SQLite 完整 replay。

## C) 工具执行契约

```ts
export type ArcCommand =
  | { k:"submit";    v:1; commandId:string; sessionId:string; input:SubmitInput }
  | { k:"interrupt"; v:1; commandId:string; turnId:string; reason:"user"|"abort" }
  | { k:"approve";   v:1; commandId:string; askId:string; decision:"allow"|"deny" }
  | { k:"declareCap";v:1; commandId:string; profile:CapabilityProfile }
  | { k:"resume";    v:1; commandId:string; sessionId:string; afterSeq:number; epoch:number };
export type SubmitInput = { text:string; agent:"code"; mode?:"chat"|"edit"|"test"; baseEpoch:number };

export type RiskClass = "read"|"write"|"irreversible"|"funds";
export type RiskTier  = "safe"|"confirm"|"admin_only";
export type ToolMeta = { name:string; description:string; isReadOnly:boolean; isConcurrencySafe:boolean;
  riskTier:RiskTier; riskClass:RiskClass; timeoutMs:number; maxResultSizeBytes:number };
export type ToolContext = { tenantId:string; userId:string; workspaceId:string; sessionId:string;
  turnId:string; callId:string; cwd:string; capability:CapabilityProfile; signal:AbortSignal;
  emit:(e:ArcEvent)=>Promise<void> };
export type Tool<In,Out> = { meta:ToolMeta; inputSchema:z.ZodType<In>; outputSchema:z.ZodType<Out>;
  execute(input:In, ctx:ToolContext):Promise<Out>; toModelOutput?(out:Out):string };

export type ToolErrorEnvelope = { status:"error"; tool:string;
  error_class:"VALIDATION"|"PERMISSION_DENIED"|"APPROVAL_DENIED"|"APPROVAL_EXPIRED"|"SANDBOX_UNAVAILABLE"
    |"SANDBOX_DENIED"|"TIMEOUT"|"CANCELLED"|"EXEC_FAILED"|"INTERNAL";
  user_message:string; retry_allowed:boolean };   // 5 键 envelope，绝不泄 traceback
```
P0 内置工具最小集：`read_file`、`write_file`、`apply_patch`、`bash`（zod schema：BashInput/BashOutput 等）。

**审批状态机**：`pending →(approve allow,未过期) allowed | →(deny) denied | →(过期) expired | →(turn 中断) cancelled`，后四态终态。策略：`safe+read` 自动允许；`confirm` 发 `permission.ask`（默认 60s 过期）；`admin_only` P0 默认拒绝（除非用户开 danger-full-access 二次确认）；黑名单永拒（读 `~/.ssh`、挂 docker.sock、访问 Keychain、输出 env secret、`sudo`、`rm -rf ~` 等）。

**完整生命周期时序**：POST /api/commands → auth 中间件解 RequestContext → 幂等检查(turns.session+commandId) → epoch 检查(baseEpoch≠sessions.epoch→StaleEpochError) → 创建 turn + turn.started → materialize tools(CapabilityProfile ∩ agentProfile ∩ permissionRules) → AI SDK streamText(message.delta 合批写 events;tool.requested) → 风险分类(shell-quote 分词+presets+黑名单+ToolMeta) → 需审批则 approvals.pending+permission.ask+暂停 → 用户 approve → SandboxService.run(local-nono) stdout/stderr→tool.progress 合批 → 超时/取消(interrupt→AbortController.abort+nono kill tree) → 超限落 artifact(spillRef) → tool.output → usage 落库 → turn.completed。

**并发规则**：只读且 concurrency-safe 工具并发上限 8；写工具串行（`bash`/`apply_patch`/`write_file`/`git` 一律视为写）；同一 session 同时只允许一个 active turn（durable queue 后置）。

**输出投影**：>32KB 落 `artifacts` 文件，事件只带 `preview`(前 16KB)+`spillRef=artifact://<id>`，模型只见 preview。

**SandboxService**（与沙箱方案一致，补事件回调 onStdout/onStderr）：`probe()/run(req)/cancel(runId)`，backend = local-nono|remote-vercel|remote-e2b|browser-pyodide|docker-fallback。

**现成 vs 薄自研**：现成=zod / AI SDK tool()+streamText() / drizzle+bun:sqlite / @modelcontextprotocol/sdk / nono / pino / simple-git / shell-quote。薄自研接缝=ArcCommand HTTP handler、appendEvent()+SSE replay、ToolRegistry(元数据/并发分批/输出投影)、ApprovalService 状态机、SandboxService adapter、ArtifactStore、EpochGuard。

## A/B/C 一致性自检
- 拓扑↔沙箱：P0 Web 连本地 Hono → `localSandbox=true`；远程 Web 才 false。
- 拓扑↔鉴权：同源 Web 用 httpOnly cookie；CLI/测试用 bearer；server.json 仅本地发现。
- 数据模型↔ArcEvent：`events(session_id,seq)` 唯一，seq per-session 单调，SSE `id:`=seq。
- 数据模型↔epoch：`sessions.epoch`=乐观锁+压缩边界；`events.epoch` 支撑 epoch-jump/resync。
- 契约↔artifacts：`tool.output.spillRef`→`artifacts.id`，模型只见 preview。
- 契约↔approvals：`permission.ask.askId`↔`approvals.id`，approve command 改状态。
- 沙箱↔secrets：`secrets_metadata` 不存明文；nono credential proxy/audit 只存引用。
- P0↔后置：不做 durable input queue / Redis resumable / AG-UI codegen / 多租户 RLS。

## P0 最小可跑切片落地顺序
1. 建表：workspaces、sessions、turns、events、messages。
2. `arclight serve --repo`：建 `.arclight/`、SQLite、server.json、loopback auth。
3. 打通 C1/C2：submit→turn.started→fake message.delta→turn.completed，SSE 可 replay。
4. 接真实 AI SDK：只接 Anthropic 一个 provider。
5. 建 ToolRegistry：先放 `read_file`、`bash`。
6. 接 nono SandboxService：`bash` 跑 `npm test`，stdout/stderr 回灌。
7. 加 approvals：bash 默认 `confirm`，Web approve 后执行。
8. 加 artifacts：stdout/stderr 超限落盘，spillRef 可下载。
9. 加 checkpoints：执行前后用 simple-git 记录 shadow-git/nono snapshot ref。
10. 加 usage + ≥10 golden eval：每个 eval 都走同一 HTTP/SSE/tool/sandbox 链路。
